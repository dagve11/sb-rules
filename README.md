# sb-rules

sing-box rule-set assets for the Nezha Agent VPN smart-routing mode.

This repository publishes CN direct rule sets:

- `geosite-cn.srs`: CN domain rules exported from SagerNet `sing-geosite`.
- `geoip-cn.srs`: CN IP CIDR rules exported from SagerNet `sing-geoip`.
- `manifest.json`: asset metadata for agents, including SHA256 and download URLs.

Agents should cache these files per policy and reference them as local rule sets in
the generated sing-box configuration. Dashboard stores only rule-set metadata and
dispatches it to the entry agent.

## Download URLs

```text
https://github.com/dagve11/sb-rules/releases/latest/download/geosite-cn.srs
https://github.com/dagve11/sb-rules/releases/latest/download/geoip-cn.srs
https://github.com/dagve11/sb-rules/releases/latest/download/manifest.json
```

## Intended Routing

Entry agents should generate route rules like this:

```json
{
  "rule_set": [
    {
      "type": "local",
      "tag": "geosite-cn",
      "format": "binary",
      "path": ".../rules/geosite-cn.srs"
    },
    {
      "type": "local",
      "tag": "geoip-cn",
      "format": "binary",
      "path": ".../rules/geoip-cn.srs"
    }
  ],
  "rules": [
    {
      "rule_set": "geosite-cn",
      "outbound": "direct"
    },
    {
      "rule_set": "geoip-cn",
      "outbound": "direct"
    }
  ],
  "final": "vpn-exit"
}
```

This means:

- CN domains: direct
- CN IPs: direct
- Everything else: `vpn-exit`

## Updating

Rules are generated by GitHub Actions:

- Daily schedule
- Manual `workflow_dispatch`

The workflow downloads upstream `geosite.db` and `geoip.db`, exports the `cn`
category with the sing-box CLI, compiles the source JSON to `.srs`, then publishes
a new GitHub release.

## Local Generation

```sh
./scripts/build-rules.sh
```

Generated files are written to `dist/`.
